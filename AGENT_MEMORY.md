# 🧠 ذاكرة المهندس (Agent Memory & Checklist)
> **الهدف:** تتبع المهام بدقة، المراجعة الذاتية، والتأكد من عدم نسيان أي قاعدة من قواعد البرومبت.

## القواعد الصارمة (للمراجعة قبل دمج أي فرع)
- [ ] **منهجية العمل الأساسية:** قم بإجراء أعمق بحث ممكن على الإنترنت لمعرفة أصح طريقة قبل البدء في أي مهمة.
- [ ] **صلاحيات GitHub:** تذكر دائماً أنك تمتلك القدرة على التحكم في GitHub، وسحب الـ PRs وقراءة تقارير وتلميحات CodeRabbit والتعامل معها برمجياً.
- [ ] **التخطيط أولاً:** لا تقم بكتابة أي كود قبل إنشاء خطة واضحة وعرضها على Nader للموافقة عليها.
- [ ] **المراجعة المستمرة:** راجع عملك وتحقق منه دائماً قبل الإبلاغ بانتهاء المهمة.
- [ ] **ذاكرة المشروع:** حافظ دائماً على هذا الملف محدثاً كذاكرة حية لتوثيق دورة حياة المشروع من البداية للنهاية.
- [ ] هل تم التنفيذ في فرع (Branch) مستقل؟
- [ ] هل تم الرفع كـ Vercel Preview؟
- [ ] هل تم عمل اختبار Regression لـ (تسجيل الدخول، Dashboard، حفظ الملاحظات، حفظ اللوحة)؟
- [ ] هل اللوحات القديمة تفتح بشكل سليم (Backward Compatibility)؟
- [ ] هل طلبنا موافقة المالك قبل دمج الفرع في `main`؟

### جلسة (25 يونيو 2026): التدقيق الشامل والمدونة
- **مراجعة CodeRabbit:** تم استقبال تقرير التدقيق البرمجي من CodeRabbit وتم حل 11 ملاحظة برمجية (تحديث الـ Shadcn، تصحيح الـ keys، الـ Firebase HMR، وإصلاح مشكلة الـ Promise في Next.js 15).
